Design Principles — v2.0 Anti-Slop Rules

These rules exist because AI-generated UI has recognisable patterns. If a design choice matches one of these patterns, redo it. The site should look like it was built by someone who gives a damn.

01
No gradient buttons

Gradient pills are AI marketing cliche. Use underline-draw + glow (btn-alive).

02
No pill-shaped buttons

border-radius: 9999px only for actual circles. Buttons use btn-alive (no border-radius at all).

03
No eyebrow labels

Small + uppercase + letter-spacing above headings is an AI slop pattern. Only use mono labels for card categorisation, never above section headings.

04
No generic fade-up animations

Use letter stagger, typewriter, sequential reveals. If it looks like default Framer Motion whileInView fadeUp, redo it.

05
No glassmorphism

No backdrop-filter outside the navigation bar. Frosted glass everywhere is an AI tell.

06
No dramatic shadows

blur > 20px only on card hover states. Dramatic ambient shadows on static elements are not used.

07
No decorative copy

Don't explain what the UI does. No "Click here to explore our offerings" or "Scroll down to learn more."

08
No transform-on-hover except btn-alive

Magnetic hover on btn-alive is intentional design. Random scale/rotate on hover elsewhere is not.

09
Colors stay flat

Not blended into gradients. Use token colours as distinct, intentional values.

10
Icon containers: cream bg + purple border

Background: #F6F2EB (cream). Border: 1.5px solid var(--primary). Not gradient backgrounds.

11
Dark CTA = hero mirror

Same letter stagger heading, same btn-alive--on-dark. Dark sections must not use different button styles.

12
Sequential, not simultaneous

Cards animate L-to-R. Content within each card reveals in order: label/icon, title stagger, typewriter desc, CTA pulse.

13
Mono labels for categorisation

IBM Plex Mono, 11px, 500 weight, 0.08em letter-spacing, uppercase, Royal Purple. Used on cards, not above sections.

14
Reduced motion is first-class

All animations have prefers-reduced-motion fallback. Content visible immediately, no motion, no cursor blink.

btn-alive System

The primary CTA pattern. Underline draws on hover via clip-path, radial glow pulses behind the text. No background, no border-radius, no gradient.

Usage Guidelines

Buttons

Do

Use btn-alive for all CTA buttons. Add --lg for hero, --sm for cards, --on-dark for dark surfaces. The underline draws on hover, the glow pulses.

Don't

Don't use gradient backgrounds, pill shapes, or filled buttons. Don't add scale/rotate on hover. Don't use a different button style on dark CTA sections.

Section Headings

Do

Use IBM Plex Serif at the section scale. On dark backgrounds, use the letter-stagger animation (StaggerHeading component). Keep headings short and direct.

Don't

Don't add an eyebrow label above section headings. Don't use generic fadeUp on dark section headings. Don't use IBM Plex Sans for section headings.

Cards

Do

Use cream bg + purple border for icon containers. Animate cards sequentially L-to-R. Use mono labels for categorisation. Hover: translateY(-4px), accent border, elevated shadow.

Don't

Don't use gradient icon containers. Don't animate all cards simultaneously. Don't use dramatic shadows on static cards. Don't add decorative text explaining the cards.

Animations

Do

Use letter stagger for headings, typewriter for descriptions, sequential reveal for cards. All animations must have prefers-reduced-motion fallback. Use --ease-out for all transitions.

Don't

Don't use generic fadeUp everywhere. Don't animate simultaneously when sequential is specified. Don't use different easing curves. Don't forget reduced-motion support.

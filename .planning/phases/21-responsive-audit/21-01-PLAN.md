Start the dev server (`pnpm dev`) in background. Then use Playwright MCP tools to systematically audit every page at each target viewport. The audit checks for visual/layout problems — not functional bugs.

**Target viewports (width x height):**
1. MacBook Pro: 1440 x 900
2. iPad landscape: 1194 x 834
3. Windows laptop: 1366 x 768
4. 4K display: 2560 x 1440
5. Mobile small: 375 x 812
6. Mobile standard: 390 x 844

**Pages to audit:**
1. Homepage (`/`) — 12 scroll-snap sections. Check EACH section individually by scrolling through them. This is the most complex page. Pay special attention to content overflow within snap sections at shorter viewports (768px, 834px, 900px heights).
2. About (`/about`)
3. AI Advisory (`/services/ai-advisory-for-leaders`)
4. AI Training (`/services/ai-training-for-teams`)
5. Fractional Exec (`/services/fractional-exec-support`)
6. Contact (`/contact`)
7. Blog index (`/blog`)
8. A blog post (pick the first available post)
9. Privacy Policy (`/privacy-policy`)
10. 404 page (`/nonexistent-page`)

**Audit procedure per page per viewport:**
1. Set viewport size using `browser_resize`
2. Navigate to the page
3. Take a screenshot or use `browser_snapshot` to inspect the accessibility tree
4. Look for these specific issues:
   - **Overflow**: Content extending beyond its container or viewport
   - **Clipping**: Text or elements cut off by parent containers
   - **Squashing**: Elements compressed to unreadable sizes
   - **Overlap**: Elements stacking on top of each other unintentionally
   - **Horizontal scroll**: Page wider than viewport causing horizontal scrollbar
   - **Text readability**: Font sizes too small at given viewport
   - **Spacing**: Excessive or insufficient padding/margin that wastes or cramps space
   - **Navigation**: Nav/dropdown usability at each viewport
   - **Footer**: Footer layout at each viewport
   - **Images**: Images not scaling or aspect ratio distortion

**For homepage specifically:**
- At each viewport, scroll through all 12 snap sections
- For shorter viewports (768px, 834px, 900px height), check if section content fits within one viewport height minus browser chrome (~50-80px). Content that overflows the snap boundary is the primary concern.
- Note which specific sections overflow and what content causes it (e.g., "Social Proof section: quote carousel + logo ticker overflow at 768px height")

**Efficiency approach:**
- Start with the most constrained viewport (Windows laptop 1366x768 — shortest height on desktop) to find overflow issues first
- Then check mobile viewports (likely different class of issues — layout/stacking)
- Then check larger viewports (4K — likely spacing/centering issues)
- Skip redundant checks if a viewport is very similar to one already tested

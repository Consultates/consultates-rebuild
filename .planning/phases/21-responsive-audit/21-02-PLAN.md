**Fix approach by issue type:**

**Homepage scroll-snap overflow (most likely bulk of issues):**
- The scroll-snap sections must fit within `100dvh`. Content that overflows the snap boundary needs responsive adjustments.
- Use `@media (max-height: 900px)` and `@media (max-height: 768px)` height-based media queries in `global.css` to reduce padding, font sizes, and gaps within scroll-snap sections at shorter viewport heights.
- Prefer `clamp()` for font sizes and padding where possible (e.g., `padding: clamp(1rem, 3vh, 3rem)`).
- If a section has too much content to fit at 768px height, consider making specific elements smaller or hiding non-essential decorative elements at that breakpoint.
- **DO NOT** change the GSAP scroll-snap system itself (ScrollTrigger config, snap durations, animation mechanics). Only adjust CSS within sections.

**Width-based responsive fixes (mobile, tablets, 4K):**
- Use standard Tailwind responsive prefixes (`sm:`, `md:`, `lg:`, `xl:`, `2xl:`) for width-based adjustments.
- For 4K: ensure content doesn't float in a tiny island — max-width containers should be reasonable but content should still be centered and readable.
- For mobile: fix stacking, font sizes, touch targets, horizontal overflow.

**Component-level fixes:**
- If a React island component needs internal responsive adjustments (e.g., fewer cards visible, smaller font), modify the component's Tailwind classes or inline styles.
- Prefer CSS-only fixes over JavaScript/React logic changes.

**Global fixes preferred over per-component fixes:**
- If multiple sections share the same overflow pattern (e.g., all need less vertical padding at short heights), add one global rule in `global.css` rather than fixing each component individually.
- Example: `.scroll-section { padding-top: clamp(1rem, 3vh, 3rem); }` fixes all sections at once.

**What NOT to do:**
- Do NOT touch GSAP scroll-snap mechanics (ScrollTrigger, snap config, animation timing)
- Do NOT change Gary's copy text
- Do NOT restructure page layouts — only adjust sizing, spacing, visibility
- Do NOT add new dependencies
- Do NOT delete any components or sections

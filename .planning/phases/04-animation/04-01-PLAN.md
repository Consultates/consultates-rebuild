Create `src/lib/` directory and `animations.ts` with these exports:

1. **fadeUp** variant — `initial: { y: 24, opacity: 0 }`, `animate: { y: 0, opacity: 1 }` with `transition: { duration: 0.6, ease: [0.25, 0.1, 0.25, 1] }`. Also export a mobile version `fadeUpMobile` with `y: 16`.

2. **scaleIn** variant — `initial: { scale: 0.95, opacity: 0 }`, `animate: { scale: 1, opacity: 1 }` with `transition: { duration: 0.5, ease: [0.25, 0.1, 0.25, 1] }`.

3. **staggerContainer** variant — `animate: { transition: { staggerChildren: 0.15, delayChildren: 0.2 } }`.

4. **hoverLift** — `whileHover: { y: -4 }` config object (not a variant, a prop spread).

5. **viewportConfig** — `{ once: true, amount: 0.3 }`. Also `viewportConfigMobile` with `amount: 0.2`.

6. **useReducedMotion** custom hook — checks `window.matchMedia('(prefers-reduced-motion: reduce)')`. Returns boolean. Uses `useState` + `useEffect` with `matchMedia.addEventListener('change')` for live updates. SSR-safe (default false, set on mount).

7. **getVariant** helper — takes a variant and reducedMotion boolean. If reducedMotion is true, returns `{ initial: {}, animate: {} }` (no transform, immediate visibility). This lets islands do `const variant = getVariant(fadeUp, reducedMotion)` instead of branching everywhere.

All variants typed with Framer Motion's `Variants` type.

- Props: `headline: string`, `paragraph: string`, `ctaHref: string`, `ctaText: string`

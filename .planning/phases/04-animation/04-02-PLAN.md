**CountUpStat.tsx** — Animated stat that counts up on viewport entry (ANIM-05):
- Props: `target: number`, `suffix?: string` (e.g. "%"), `label: string`, `duration?: number` (default 600ms)
- Uses `framer-motion` `useMotionValue`, `useTransform`, `animate` from `framer-motion`
- Uses `useInView` ref to trigger animation when element enters viewport (`once: true, amount: 0.3`)
- On viewport entry: animate motionValue from 0 to `target` over `duration`ms
- `useTransform` to round the value to integer for display
- Display: Render the stat number in `text-stats text-foreground font-serif font-bold` with suffix appended
- Below it render `label` in `text-body text-muted-foreground mt-2`
- If `useReducedMotion()` returns true: display final value immediately, no animation
- Ensure `client:visible` compatibility (component mounts when scrolled into view)

**StakesCTAPulse.tsx** — CTA button with continuous pulse animation (ANIM-06):
- Props: `href: string`, `text: string`
- Renders a `motion.a` tag styled exactly like the existing Stakes CTA anchor (same classes from StakesCTASection.astro scoped styles, but as inline/Tailwind: `inline-flex items-center justify-center font-sans font-semibold rounded-lg h-12 px-8 text-body-lg transition-all duration-150 hover:scale-[1.02]` with `style={{ background: '#5C3B9C', color: '#FFFFFF' }}`
- Initial animation: scaleIn variant with 600ms delay (waits for headline fadeUp)
- After initial animation completes (use `onAnimationComplete`): start infinite pulse
- Pulse: `animate={{ scale: [1, 1.05, 1] }}` with `transition={{ duration: 0.4, ease: 'easeInOut', repeat: Infinity, repeatDelay: 2 }}` — pulses once every ~2.4s
- Hover: `whileHover={{ scale: 1.02, background: '#4E3285' }}`
- If `useReducedMotion()`: no pulse, no scaleIn, just static button

**SectionAnimator.tsx** — Reusable wrapper for section entrance animations:
- Props: `children: ReactNode`, `variant?: 'fadeUp' | 'stagger'`, `delay?: number`, `className?: string`, `as?: 'div' | 'p' | 'h2'` (default 'div')
- If `variant === 'stagger'`: uses `staggerContainer` as parent variant, children get fadeUp
- If `variant === 'fadeUp'` (default): wraps in `motion.div` with fadeUp variant + delay
- All use `whileInView="animate"` with `viewportConfig` (once, amount 0.3)
- If `useReducedMotion()`: render children immediately in static wrapper, no motion component
- This is used by Astro sections via `client:visible` to wrap their inner content

IMPORTANT: Every React island in non-hero sections uses `client:visible` (NOT `client:load`). Only Hero uses `client:load`.

IMPORTANT: Astro components (ServiceCard, TestimonialBlock, StepCard, BlogCard, StatBlock) cannot be children of React components. SectionAnimator wraps around the Astro component's OUTPUT — meaning wrap the Astro component call itself, and SectionAnimator's children are passed via slot which renders the server-rendered HTML. This works because `client:visible` hydrates the React wrapper, and the Astro component's HTML is already inside it as children.

Actually — Astro components CANNOT be passed as children to React islands. The correct pattern: Use `<SectionAnimator client:visible><div>` and put the static HTML content directly inside, or use `set:html` with the rendered output. The SIMPLEST correct approach: For sections with Astro sub-components (ServiceCard, StepCard, etc.), do NOT wrap individual cards. Instead, wrap the entire section's inner content div in a single SectionAnimator. For simple text sections, wrap text elements directly.

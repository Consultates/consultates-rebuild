Add a `<script>` tag (NOT `is:inline` — use Astro's standard script bundling so GSAP gets tree-shaken) at the bottom of the `<body>` in Base.astro, after `<Footer />`:

```typescript
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.registerPlugin(ScrollTrigger);

// Gate: only run on desktop AND no reduced motion
const isDesktop = window.matchMedia('(min-width: 1024px)').matches;
const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

if (isDesktop && !prefersReducedMotion) {
  const sections = gsap.utils.toArray<HTMLElement>('.scroll-section');

  sections.forEach((section) => {
    // Zoom transition: start scaled down and transparent, animate to full
    gsap.fromTo(section,
      { scale: 0.92, opacity: 0 },
      {
        scale: 1,
        opacity: 1,
        duration: 0.5,
        ease: 'power2.out',
        scrollTrigger: {
          trigger: section,
          start: 'top bottom',
          end: 'top top',
          scrub: false,
          toggleActions: 'play none none reverse',
        }
      }
    );
  });

  // Don't apply zoom to first section (hero) — it should be visible on load
  // Kill the ScrollTrigger on the first section
  if (sections.length > 0) {
    gsap.set(sections[0], { scale: 1, opacity: 1 });
    ScrollTrigger.getAll().find(st => st.trigger === sections[0])?.kill();
  }

  // Scroll snap behavior
  ScrollTrigger.create({
    snap: {
      snapTo: 1 / (sections.length - 1),
      duration: { min: 0.3, max: 0.6 },
      ease: 'power2.inOut',
    },
  });

  // Scroll indicator fade-out: hide when #section-trust enters
  const scrollIndicator = document.querySelector('[data-scroll-indicator]');
  if (scrollIndicator) {
    ScrollTrigger.create({
      trigger: '#section-trust',
      start: 'top 80%',
      onEnter: () => gsap.to(scrollIndicator, { opacity: 0, duration: 0.3 }),
      onLeaveBack: () => gsap.to(scrollIndicator, { opacity: 0.6, duration: 0.3 }),
    });
  }
}
```

IMPORTANT: The HeroIsland.tsx scroll indicator element needs a `data-scroll-indicator` attribute so GSAP can target it. This was set up in Plan 01 — verify it exists. If not, note in the plan that the executor should add it.

Note: The GSAP script runs in `<script>` (not `is:inline`) so it's bundled by Astro/Vite. GSAP is already in package.json. The `gsap/ScrollTrigger` import works because GSAP v3.12 ships ESM.

Also ensure the first section (hero) is NOT affected by zoom-in — it loads at full scale since it has its own Framer Motion entrance. Set `gsap.set(sections[0], { clearProps: 'all' })` after killing its ScrollTrigger, or better: start the forEach loop from index 1 instead of 0.

Revised approach — simpler and cleaner: Loop from `sections.slice(1)` to skip hero entirely. Hero needs no GSAP treatment.
